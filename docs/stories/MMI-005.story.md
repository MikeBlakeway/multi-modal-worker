```python
# Optimized FLUX.1 model loading with memory management
def load(self) -> None:
    self.pipeline = FluxPipeline.from_pretrained(
        self.model_path, torch_dtype=torch.float16, variant="fp8"
    )
    # Memory optimizations for 12-15GB target
    self.pipeline.enable_sequential_cpu_offload()
    self.pipeline.enable_attention_slicing()
```

## Details

### Problem

The multi-modal inference worker needs to implement text-to-image generation using FLUX.1 Schnell fp8 model as the primary image generation capability. This is the most fundamental modality and serves as the foundation for other image-based modalities like ControlNet and inpainting.

### Impact

Without FLUX.1 text-to-image implementation:

- Core image generation functionality will be unavailable
- Users cannot generate images from text prompts
- Foundation for other image modalities (ControlNet, inpainting) will be missing
- The system cannot demonstrate basic AI inference capabilities
- Performance benchmarking and optimization cannot begin

### Scope

This story covers implementing the complete FLUX.1 Schnell handler including model loading, inference pipeline, parameter validation, and output formatting. Requires MMI-003 (Model Management) and MMI-004 (Handler Routing) to be completed first.

---

## Acceptance criteria

- Given a text prompt, when text-to-image inference is requested, then FLUX.1 generates a high-quality image within 15 seconds.
- Given image parameters (width, height, steps), when provided, then the generated image matches the specified dimensions and quality settings.
- Given invalid parameters, when validation fails, then clear error messages are returned explaining the parameter constraints.
- Given memory constraints, when FLUX.1 is loaded, then it integrates with the model management system for efficient memory usage.
- Given concurrent requests, when multiple text-to-image jobs run, then each completes successfully without interference.
- All generated images must be properly encoded and returned in the standardized response format.
